     consumer — do not modify any backend file. -->

`POST /runs` -> `{"run_id": string, "status": "awaiting_review" | "running", "plan": Plan | null, "push_report": PushReport | null}`

`GET /runs/{run_id}` -> `{"run_id": string, "status": "running" | "awaiting_review" | "completed" | "not_found", "plan": Plan | null, "push_report": PushReport | null}`

`POST /runs/{run_id}/resume` body `{"approved": boolean}` -> same shape as GET, reflecting post-resume state.

Plan shape (mirror as a TypeScript type in runClient.ts — do not import Python types, this is a
parallel TS declaration of the same JSON contract per CLAUDE.md's "shared shape" rule applied at the
API boundary, not a second source of truth for backend logic):

```ts
interface Task {
  id: string;
  title: string;
  description: string;
  suggested_assignee: string;
  estimate_hours: number;
  skill_tag: string | null;
  depends_on: string[];
}
interface Epic {
  id: string;
  title: string;
  description: string;
  tasks: Task[];
}
interface Plan {
  epics: Epic[];
}
interface PushResultItem {
  item_id: string;
  ado_work_item_id: number | null;
  status: "created" | "assignment_unresolved" | "create_failed" | "link_failed" | "not_implemented";
  detail: string | null;
}
interface PushReport {
  items: PushResultItem[];
  all_succeeded: boolean;
}
interface RunResponse {
  run_id: string;
  status: "running" | "awaiting_review" | "completed" | "not_found";
  plan: Plan | null;
  push_report: PushReport | null;
}
```
</interfaces>

## Trust Boundaries

|----------|--------------|
| Browser (React dev server) -> FastAPI backend | Unauthenticated local HTTP fetch calls to POST /runs, GET /runs/{id}, POST /runs/{id}/resume |
| Frontend-rendered plan/push_report content | Backend-controlled JSON rendered as plain text/lists in the DOM — no user-supplied input reflected in this phase (D-07/D-08 has no editable fields) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|------------------|
| T-01-11 | Tampering | Unauthenticated frontend can call resume on any run_id it knows | accept | Explicit MVP scope: no auth, single local lead, single browser session per CLAUDE.md's non-negotiable constraint; run_ids are server-generated uuid4 hex, not guessable in practice, and the risk surface is a local-only tool with no external network exposure |
| T-01-12 | Information Disclosure | CORS configuration (if hardcoded base URL approach chosen in Task 1) permits localhost origins broadly | mitigate | Scope CORS allow-list to localhost dev origins only (e.g. http://localhost:5173, Vite's default port) if backend CORS is touched; do not use a wildcard `*` origin even for local dev, to avoid establishing a bad habit that could survive into a less-local future |
| T-01-13 | Spoofing / DoS | No timeout on frontend fetch calls | accept | Low risk for a local-only 2-day MVP polling a local backend; acceptable to defer explicit fetch timeout/abort-controller handling past Phase 1 |
