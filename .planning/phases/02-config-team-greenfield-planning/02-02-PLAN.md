<objective>
Build the team roster as a complete, independent vertical slice: a
`team_members` SQLite table, a `TeamMember` Pydantic model, FastAPI CRUD
routes, and a minimal React page wired end-to-end — the lead can add, edit,
and remove team members before planning starts (TEAM-01/TEAM-02).

Purpose: The roster is intentionally decoupled from the LangGraph run state
(per research's Architectural Responsibility Map: "independent of the
LangGraph run — never part of RunState"). This plan delivers that whole slice
in isolation so it can be built and verified in parallel with Plan 01's
config/smoke-test work, with zero file overlap.
Output: A working `/team` CRUD surface (backend + frontend), reachable and
demoable independent of any run being started.
</objective>
